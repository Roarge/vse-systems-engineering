## 2. System Requirements

<!-- VSE-TASK: SR.2.4 -->

> Each requirement must satisfy the SMART criteria:
> - **S**pecific: unambiguous, single interpretation
> - **M**easurable: quantifiable acceptance criterion
> - **A**chievable: technically feasible within constraints
> - **R**elevant: traceable to at least one stakeholder need
> - **T**ime-bound: verifiable within the project timeline
>
> Corresponding SysML 2.0 models go in `models/system-requirements.sysml`
> with IDs using the REQ- prefix and satisfy links to STK- needs.
